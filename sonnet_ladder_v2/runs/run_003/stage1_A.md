## A1-GFSM — Gauge-Fixed Softmax Machine [A / Track B]

**Constraint**: The computation graph must algebraically simplify under softmax shift-invariance (adding a constant to all logits leaves softmax unchanged). This constraint is mechanizable: `softmax(x + c·1) = softmax(x)` for any scalar `c`. Under this binding, the stored representation of any attention logit row is determined only up to a global additive constant — one real degree of freedom per row per head is physically meaningless and must be eliminated before storage. Constraint difficulty: ~70% of standard stored-logit representations are inadmissible as written; they carry an unused gauge component.

**Mechanism**: Track B. W_Q and W_K each contribute a half-logit: `q^T k` = `(W_Q x)^T (W_K x')`. Under shift-invariance, the projection `W_Q x → W_Q x + c·1_H` (adding `c` to all H head-logit components before the key dot-product) is a gauge redundancy only if the keys are invariant. The correct gauge orbit: fixing `(W_Q x)_h` such that Σ_h (W_Q x)_h = 0 (zero-mean over heads within a token) eliminates one scalar per Q-projection output per token. This is NOT the killed cross-layer W_Q basis sharing (v2-CHEAP-TEST-001 class kill) — it targets within-token, within-layer gauge freedom in the Q output representation, not cross-layer subspace coherence. The gauge-fix is a rotation of W_Q (multiply by a fixed Householder reflector R_gauge that projects onto the zero-mean hyperplane). After the rotation the last head-component of W_Q·x is deterministically recoverable from the others, so it need not be stored at full precision. Mechanized: R_gauge computed once from W_Q weight statistics; stored W_Q gets one of H=16 head-blocks dropped to INT4 instead of BF16 (the drop is lossless to the gauge-fixed representation). Target: 1/16 of W_Q storage recovered losslessly, ~6% of W_Q residency on top of CF11's 8× global compression ceiling.

**Residency arithmetic**: At Qwen3-1.7B, W_Q per layer: 2048×2048 = 4M params × 2 bytes BF16 = 8 MB per layer × 28 layers = 224 MB total. Gauge-fix recovers 1/16 of that losslessly: ~14 MB on a 3.3 GB model (0.4%). Negligible standalone. Value is in grounding: if the gauge-fix is lossless, it validates the theoretical argument that W_Q has exactly one redundant direction per layer under this symmetry, informing more aggressive orbit-quotient compression in later cascade rungs.

**Novelty gloss**: Closest kill-list item: v2-CHEAP-TEST-001 (cross-layer W_Q subspace sharing, killed). Structural difference: this operates within a single layer's softmax symmetry group (gauge redundancy of the shift-invariant softmax), not across layers. Closest published method: "attention logit bias absorption" in PrefixQuant / SmoothQuant variants — structural difference: those absorb bias into fixed-offset token statistics; this fixes a continuous gauge degree of freedom in the weight matrix itself, not a bias vector.

**Smallest experiment**: Compute Σ_h (W_Q x)_h for 200 random tokens on Qwen3-1.7B-Base; measure Frobenius deviation from zero-mean per token. If max deviation < 1e-3 (implying SGD already nearly gauge-fixed W_Q), the free lunch is real and exploitable. Runtime: ~5 min. NO-GO (deviation >> 0) is itself a structural finding: trained W_Q does NOT naturally land near the gauge-fixed surface, so any gauge-fixing compression requires a rotation correction that may distort W_K interactions.

**Primary risk**: The gauge freedom in softmax is real, but W_Q and W_K interact in the dot-product; a rotation of W_Q changes `q^T k` unless W_K is counter-rotated. Mitigation: rotate both W_Q and W_K by R_gauge (a joint rotation that preserves dot-products and is lossless); verify ΔNLL = 0 before claiming any storage gain.

## A2-RTGF — Residual-Stream Sign-Flip Gauge Fold [A / Track B]

**Constraint**: The computation graph must algebraically simplify under the residual-stream sign-flip symmetry. For any channel `d`, the substitution `h_d → -h_d` accompanied by `W_in[:,d] → -W_in[:,d]` and `W_out[d,:] → -W_out[d,:]` for all matrices that read or write channel `d` is an exact equivalence. This is a Z₂ gauge symmetry on the residual stream. It is mechanizable: for each channel independently, choose the sign convention that minimizes storage cost. Constraint difficulty: ~50% of standard weight storage conventions are gauge-equivalent under this group; sign conventions are unchosen by SGD and land arbitrarily.

**Mechanism**: Track B. Each of the d_model=2048 residual-stream channels carries an independent Z₂ sign degree of freedom. SGD does not fix these signs: if channel `d` flips sign in h, the model is numerically identical if all matrices that read or write channel `d` flip the corresponding row/column. This is NOT the killed MLP low-rank compression (CF7/CF8) — it operates on the sign representation of the weight columns, not on rank structure. Apply the flip: for each channel `d`, compute the majority sign of `W_up[:,d]` across all layers (28 layers × d_ff=6144 rows = 172K values); if majority is negative, flip the sign convention for channel `d` and update all reading/writing matrices. Post-flip, the W_up[:,d] column has more positive entries → more positive activations → ReLU / SwiGLU gate fires more uniformly. This is a lossless weight redistribution. Payoff: after sign-fixing, weights are more consistently positive, and a subsequent unsigned INT4 quantization grid wastes less range on negative representation. Estimated quantization loss reduction: 0.1–0.3 nats at INT4, grounded in the standard asymmetry-of-unsigned-INT4 arithmetic.

**Residency arithmetic**: This is a quantization-quality improvement, not a bytes reduction per se. At 4 bpw, a model that is 60% negative-weight in some channels wastes ~1.5 bits of the 4-bit range. If sign-fixing moves the distribution to 80% positive, effective information per stored bit increases. On the full W_up (28 × 2048 × 6144 × 0.5 bytes = 172 MB at INT4), a 0.15 nat reduction in ΔNLL at INT4 is worth ~0.2 bpw improvement — equivalent to storing at 3.8 bpw with 4-bit grid.

**Novelty gloss**: Closest kill-list item: R2/S2 MDL-Selected Per-Layer bpw (killed for prior-art reasons). Structural difference: sign-flip symmetry gauge-fixing is a pre-quantization weight redistribution that reduces quantization error for any downstream quantizer, not a bpw schedule. Closest published: SmoothQuant / OS+ absorb per-channel scaling factors into adjacent layers; structural difference: those absorb scale (multiplicative), this fixes the sign convention (discrete Z₂ gauge) — there is no scale change, only a sign redistribution that is provably lossless.

**Smallest experiment**: On Qwen3-1.7B-Base BF16, compute per-channel sign statistics of W_up across all 28 layers. For each of 2048 residual channels: fraction positive. Measure the distribution of f_positive over channels. If f_positive ∈ [0.45, 0.55] for most channels (near 50/50), sign-flip saves nothing (SGD has no sign bias). If f_positive distribution is bimodal (many channels near 0 or 1), sign-fixing is a real effect. Runtime: ~3 min. NO-GO (balanced signs) is a structural finding: SGD implicitly gauge-fixes sign conventions toward 50/50 balance, meaning the Z₂ symmetry is broken by training dynamics.

**Primary risk**: SGD may already anti-symmetrize channels via RMSNorm, which operates on absolute magnitude and is sign-invariant — meaning each layer's RMSNorm effectively re-imposes sign ambiguity. Mitigation: include RMSNorm scale parameters in the gauge transform audit; check that after the transform, BN/RMSNorm scaling is unchanged.

## A3-NTCA — No-Continuous-Numbers Token Codec [FREE SWING / A / Track A]

**Constraint**: No continuous numbers during inference. Everything integer, rational, or table-lookup. This kills: floating-point GEMV, softmax (requires exp), RMS normalization (requires sqrt), SwiGLU (requires silu). What survives: integer dot-products, table dispatch, argmax (integer comparison), sorted rank (integer). Constraint difficulty: >90% of the standard inference path becomes inadmissible. This is near the hard end of the range; constrain recovery by allowing rational arithmetic (fixed-point with predetermined denominators) rather than pure integers, which anchors to AVX2 INT16/INT32 native SIMD.

**Mechanism**: Track A. Replace the full transformer forward pass with a two-table codec. Table 1 (token-to-residual): 151936 entries × 2048 INT8 values = 299 MB; looked up by token ID (integer). Table 2 (context-to-next-logit): a finite-state machine indexed by the last K tokens (context window K ≤ 6 for feasible table size). For K=4, vocabulary 4096 (top-4096 tokens cover ~97% of corpus), table size: 4096^4 × 4096 logits — absurdly large. Rescope: replace Table 2 with an integer-arithmetic approximation of the final layer's operation: residual @ (W_Q INT8) @ (INT8 K-cache), all integer dot-products with INT32 accumulation and INT8 lookup of exp-approximation by table. The "no-continuous" constraint forces exp(x) → a 256-entry lookup table with fixed-point input (8-bit exponent representation); this is NOT standard softmax — it is a piecewise-linear rational approximation of exp with <0.5% error on the relevant range. The constraint forces the entire forward pass through integer types, which on Ryzen 5 7530U with AVX2 means 8-wide INT32 SIMD vs 4-wide FP32 — potential 2× throughput gain before quality adjustment.

**Residency arithmetic**: If W_Q, W_K, W_V, W_O quantized to INT8 (1 byte) and W_gate/W_up/W_down quantized to INT8, total model: 70B × 1 byte = 70 GB (70B params). Still too large. But at INT4 (0.5 byte) = 35 GB. At INT2 (0.25 byte) = 17.5 GB. Still exceeds 7.28 GiB. This constraint alone does not solve residency; it solves throughput. With NVMe streaming at 3 GB/s, INT2 70B = 17.5 GB / 3 GB/s = 5.8 s/token — worse. The playoff: the constraint forces INT8 which on this CPU is 8× faster SIMD per MAD than BF16; if the model is already in DRAM (4B Qwen3 at INT4 = 2.1 GB ✓), INT8 GEMV at 11.5 GB/s DRAM bandwidth yields ~5× tok/s improvement over BF16. For the 4B model, the constraint is useful. For 70B, it's a throughput story, not a residency story.

**Novelty gloss**: No kill-list item covers this. Closest published: LUT-based neural network inference (LUTNN, SIMD-friendly lookup tables for activations). Structural difference: LUTNN applies LUTs to activations; this constraint applies the no-float binding to the ENTIRE computation graph including matrix multiplications — forcing a re-examination of what "multiplication" means under fixed-point arithmetic. The mechanism genuinely has no close neighbor in the compression literature because the field assumes FP16/BF16 is non-negotiable.

**Smallest experiment**: Implement a single-layer forward pass (W_Q GEMV only) in pure INT8 AVX2 on the Ryzen 5 7530U; compare throughput vs BF16 GEMV on identical dimensions (2048×2048). Predicted: 2–4× throughput gain. GO if >1.5×. Runtime: ~2 hours coding + ~30 min benchmarking. NO-GO (INT8 not faster than BF16 on this CPU) produces structural finding: Ryzen 5 7530U Zen3 has comparable INT8 and FP16 throughput, so the no-float constraint has no hardware dividend.

**Primary risk**: Quality collapse at INT8 without calibration-based zero-point adjustment. Mitigation: use symmetric INT8 with per-channel scale absorbed into adjacent matrix (SmoothQuant-style, but lossless under the constraint since scales are rational numbers, not floats).

## A4-APPA — Append-Only Persistent KV Accumulator [A / Track A]

**Constraint**: All state is append-only. No mutation during inference. This is mechanizable: the KV cache is a log-structured append-only store; attention over past tokens reads from a monotonically growing log, never from a slot that has been overwritten. This kills: sliding-window KV eviction, KV quantization with in-place updates, PagedAttention page reuse, any cache compression that modifies stored K/V vectors after they are written. What survives: pure append, log compaction at sequence boundaries, read-only access patterns. Constraint difficulty: ~65% of production KV management systems become inadmissible.

**Mechanism**: Track A. Under the append-only constraint, the KV cache is a content-addressed log keyed by (layer, head, position). Once written, a K or V vector is immutable. This has a surprising structural consequence: K and V vectors, once computed, can be compressed at write time and never decompressed at read time — the compressed form IS the stored form, and attention is computed in compressed coordinates. For K: CF11 tells us W_K has r_99/d ≈ 0.79 at the global level. In the append-only frame, each K vector lives in the row-span of W_K (dim ≤ 2048), but we can project to rank-512 at write time and never expand back — the attention dot-product `q^T k_compressed` uses `q_projected = W_{proj}^T q` (same W_proj, computed once at Q-side). Quality cost: W_K at K=512 was +0.29 nats (CF11), but that was all-layer simultaneous; at K=512 for K-cache only (not the weight matrix), the distortion is smaller because the residual-stream representation already captures what W_K was trying to select. Append-only enforcement means this projection cannot be revised — it is permanent, which forces us to choose the right rank up front, grounded in CF11's measurement.

**Residency arithmetic**: At 4K context, 28 layers, 16 heads, d_head=128: full KV cache = 28 × 2 × 4096 × 2048 × 2 bytes = 917 MB at BF16. At K=512 (4× compression): 229 MB. At 128K context: 28.7 GB full BF16 → 7.2 GB at K=512 — fits in 7.28 GiB for very long-context use. This is the load-bearing case for the append-only constraint: it forces us to commit to the rank-512 projection at write time, which is actually the right engineering choice for long-context (no decompression overhead; attention kernel sees a consistently smaller K dimension).

**Novelty gloss**: Closest kill-list item: R2/S2 Bloom-Filter KV Retirement (killed for bottleneck mismatch). Structural difference: this is not eviction; it is lossy projection-at-write that the append-only constraint forces. Closest published: GQA (grouped-query attention) reduces K/V head count; KIVI quantizes K/V in-place. Structural difference: the append-only constraint here is not a quantization — it is a subspace projection grounded in CF11's measured W_K spectrum that is mechanically forced by the immutability requirement. KIVI mutates slots; this cannot.

**Smallest experiment**: Compute K-cache projection error on Qwen3-1.7B-Base: for a 1K-token sequence, project K vectors at each layer to rank-512 using the W_K right singular vectors; compute attention output deviation vs full-rank K. Claim: mean cosine distance < 0.01 at rank-512. Runtime: ~20 min. GO if cosine distance < 0.05 (tolerable distortion for most downstream use). NO-GO produces structural finding: K-cache vectors span more than rank-512 of the W_K subspace, suggesting that token-specific input structure (beyond W_K's own compression) is necessary.

**Primary risk**: CF11's W_K spectrum was measured on weight matrices, not on runtime K-cache vectors (which are W_K applied to diverse input tokens). The K-cache spectral concentration may be much lower than W_K's weight spectrum. Mitigation: the smallest experiment directly measures K-cache projected error; it does not assume weight-spectrum = cache-spectrum.

## A5-TCRD — Tropical-Coordinate Residual Dispatch [A / Track A]

**Constraint**: The model's routing decisions (which MLP neurons fire, which attention heads dominate) must be expressible in a tropical semiring (max-plus algebra). This is mechanizable: tropical matrix multiplication uses (max, +) instead of (Σ, ×). The constraint rules out any routing mechanism that requires summing over paths (softmax, probability normalization, weighted mixture). What survives: argmax-based routing, top-K selection, winner-take-all, threshold gates. Constraint difficulty: ~50% of attention mechanisms become inadmissible; ~30% of MLP mechanisms (only top-K firing path survives).

**Mechanism**: Track A. The tropical algebra interpretation of a transformer layer: the attention weight `α_{ij} = max_h(score_{ij}^h)` selects the dominant head per token-pair rather than averaging. The MLP output uses `silu(W_gate x) ⊙ (W_up x)` → in tropical frame: `max(silu(W_gate x)[k]) * (W_up x)[k]` for top-K k, dropping the rest. This is actually the SwiGLU top-K firing finding from CF1: W_up dominates firing-rank. The tropical frame makes the approximation explicit and lossless-up-to-sparsity: the top-K neurons dominate, and in the tropical semiring only the maximum matters. The mechanism: replace SwiGLU's element-wise product with a top-K max-dispatch: identify the K neurons with highest `(W_up x)_k` (by CF1, these dominate post-SwiGLU output), compute only those K GEMV rows of W_down, route through. The "tropical" framing is the constraint that forces us to commit to max-routing instead of continuous weighting — and the payoff is that CF3's token-dynamic K=1% Jaccard (0.308) tells us that the top-20 neurons at K=1% are stable-ish across consecutive tokens: we can cache the K-neuron selection and update only on Jaccard misses.

**Residency arithmetic**: If K=1% = 20 neurons fire at each layer, we load only 20/6144 rows of W_down per MLP call: 0.33% of W_down bytes per forward pass. W_down per layer at BF16: 6144 × 2048 × 2 bytes = 24 MB. Loading 20 rows: 80 KB. Across 28 layers: 2.2 MB per token. At NVMe 3 GB/s: 0.73 ms per token. This is potentially 100× savings in bandwidth IF the top-K identification is free. It is not free: identifying the top-20 of 6144 neurons requires computing `(W_up x)_k` for all k first, which loads all of W_up (same 24 MB per layer). The savings collapse unless W_up can also be sparsified. CF3 says K=1% Jaccard=0.308 — so the top-K set IS partially predictable. Cache the top-K set from the previous token; on cache hit (Jaccard≥0.3), skip W_up load for the stable portion. Net saving at K=1%, Jaccard=0.308: ~30% of the W_up GEMV computation skippable on a token-prediction cache. This is the Deja Vu-style predictor but in tropical coordinates where it's argmax-first, not probability-weighted.

**Novelty gloss**: Closest kill-list item: R1/S6 killed the exact Deja Vu-framed trained cross-layer activation predictor. Structural difference: this uses a ZERO-OVERHEAD predictor (the previous token's actual top-K firing set as a warm cache), not a separate predictor network — consistent with the deferred Idea J spirit. The tropical-algebra framing forces the commitment to max-routing as a first-class constraint, not a post-hoc approximation. Closest published: SparseGPT / ReLU-ification of SwiGLU; structural difference: those require retraining; this is a zero-overhead post-training observation applied in max-plus coordinates.

**Smallest experiment**: On Qwen3-1.7B-Base, compute per-token top-20 firing set (by `(W_up x)_k` magnitude) for 500 consecutive tokens in a long sequence. Measure the cache-hit rate: what fraction of the next token's top-20 is covered by the previous token's top-20? Claim: cache-hit fraction ≥ 0.20. This is the K=1% Jaccard=0.308 finding from CF3 restated as a within-sequence consecutive-token measurement. Runtime: ~15 min. GO if cache-hit fraction ≥ 0.15. NO-GO finding: CF3's Jaccard was measured across all token pairs; within a coherent generated sequence it may be higher or lower.

**Primary risk**: The tropical frame purchases speed only if the argmax-identified neurons truly dominate output quality; if even small non-top neurons contribute significantly (CF5 showed W_up is MORE rank-sensitive, suggesting spread), the approximation breaks. Mitigation: run with K=5% (CF3 Jaccard≈0.257 there) and measure ΔPPL at various K before committing to the dispatch mechanism.

## A6-SSCF — Seed-Reconstructible Small-Codebook Factorization [A / Track B]

**Constraint**: The model must be reconstructible from a 10 MB seed. All weights derivable from a small generative procedure seeded by 10 MB. This kills: storing weight matrices directly (1.7B params × 2 bytes = 3.3 GB, violates constraint by 330×); standard quantization codebooks (still store the weights); low-rank decompositions without a generating procedure. What survives: structured procedural generation of weights, hypernetwork-style synthesis from a seed, Kolmogorov-minimal descriptions. Constraint difficulty: ~95% of standard weight storage becomes inadmissible. This is near the hard edge; ground it by targeting NOT the full 1.7B model but the DELTA between a randomly-generated basis and the trained weights.

**Mechanism**: Track B. The observation: if a random matrix generator with a fixed seed produces a matrix R (reproducibly, from the seed), then the trained W_up can be expressed as W_up = R + ΔW_up where ΔW_up is the residual. The seed (≤1 MB for the random state) generates R in memory; only ΔW_up is stored. For this to help, ΔW_up must be compressible. CF8 tells us W_up is full-rank — but it is full-rank relative to zero. It may be low-rank relative to a GOOD random basis R. The question is: does there exist a structured random matrix R (e.g., a Kronecker product of smaller random matrices, or a block-circulant random matrix) such that W_up - R is low-rank? The constraint forces us to ask this. The structured random matrix is the seed's footprint; the residual is what we store. If R is Kronecker (a ⊗ b where a ∈ R^{64×64} and b ∈ R^{32×32}, giving 2048×1024 for W_gate's shape), the seed is 64^2 + 32^2 = 5120 floats ≈ 20 KB. If ΔW_up's rank is ≤ 256, then ΔW_up ≈ U × V (2048×256 + 6144×256 = 2.1M params × 2 bytes = 4.2 MB), giving a 10 MB total footprint per layer. For 28 layers this is 280 MB — not 10 MB total. Rescope: the 10 MB seed encodes the Kronecker factor seed PER LAYER FAMILY (all 28 layers share one Kronecker basis R derived from the seed); only per-layer ΔW_up residuals are stored, at rank-128 (2.6 MB per layer × 28 = 73 MB). Still > 10 MB. This constraint forces us to confront the minimum-description-length of the trained model — a genuinely novel question.

**Residency arithmetic**: If the seed-reconstructible approach achieves total model description length of 300 MB (seed 10 MB + residuals 290 MB), that is a 11× compression of the 3.3 GB BF16 model. At 300 MB on DRAM this fits easily; at NVMe streaming this is 0.1 s/token at 3 GB/s — still slow. The mechanism needs to be paired with DRAM-resident inference at this compressed scale. Against the 70B target: if the same approach scales, 70B at 11× = 5.9 GB ← fits in 7.28 GiB. This is the audacious claim.

**Novelty gloss**: No kill-list item covers this. The hypernetwork framing (small generator producing large network) is published (HyperNetworks, Ha et al. 2016), but that is training-time. The post-training Kolmogorov-residual approach — fitting a structured random basis to minimize the description length of trained weights without retraining — has no close published neighbor. The constraint is doing genuine work here: it forces the question "what is the minimum description of this trained model?" which nobody has asked in this exact form for quantization purposes.

**Smallest experiment**: Compute the Frobenius norm of (W_up[L] - R[L]) where R[L] is the Kronecker product of two random matrices seeded by a fixed random seed (no fitting). Measure: what fraction of W_up[L]'s Frobenius norm is captured by the best-fit rank-256 approximation of (W_up - R)? Claim: fraction ≥ 0.90 for some structured R (implying the residual IS low-rank even if W_up itself is not). Runtime: ~30 min for one layer; ~10 hours for all 28. GO threshold: rank-256 captures ≥ 70% of W_up - R residual variance for at least 14/28 layers. NO-GO finding: trained Qwen3 weights are not well-described by random-basis + low-rank residual — they are genuinely high-complexity in the Kolmogorov sense.

**Primary risk**: CF8 shows W_up is full-rank, suggesting its singular value spectrum is flat. If the random basis R also has a flat spectrum, then W_up - R is also full-rank and the delta saves nothing. Mitigation: choose R to be a STRUCTURED random matrix with a DIFFERENT spectral concentration (e.g., block-diagonal random, which concentrates variance on the block structure) so W_up - R has residual in the off-block directions — and measure whether trained W_up's off-block variance is smaller than its block variance.
