## A1-PCAG: Per-Channel Accumulator Gate — Integer-Only Residual Stream

**Constraint:** No floating-point arithmetic during the residual accumulation steps — the running residual sum is maintained as a per-channel int32 accumulator throughout all 28 layers.

**Track A.** Standard inference keeps the residual stream in bf16 or fp16: $h_{L+1} = h_L + \Delta_L$. Under the int32 constraint, each layer's contribution $\Delta_L$ is quantized to int16 before addition; the running sum is a per-channel int32 (no overflow for 28 layers of int16). RMSNorm reads the int32 accumulator, applies per-channel fp32 scale γ and fp32 reciprocal-RMS normalization, then outputs fp32 to the attention/MLP sub-layer. Constraint difficulty: eliminates all bf16 residual accumulation, all fused layer-norm-plus-matmul kernels that assume fp16 state (~70% of existing inference kernels). What survives: integer accumulation paths (quantized inference stacks like ExLlamaV2, llama.cpp with GGUF integer offsets), plus a new structural question about accumulator overflow headroom. The int32 accumulator per token per channel is 28 × 2048 × 4 bytes = 224 KB — fits in L2 (512 KB on Zen 3). Every layer reads from L2 (int32 accumulator) rather than DRAM (bf16 residual), saving 2048 × 2 bytes = 4 KB DRAM round-trip per layer per token. At 28 layers: 28 × 4 KB = 112 KB of residual-stream traffic eliminated per token. At 11.5 GB/s DRAM bandwidth: 112 KB / 11.5 GB/s ≈ 9.7 µs/token saved — negligible. Real win: accumulator held in L2 means residual reads/writes are at ~300 GB/s L2 bandwidth, not 11.5 GB/s DRAM. For 28 layers: 28 × 2048 × 4 bytes = 224 KB read + write per token from L2 → throughput-limited by matmul weight fetch, not residual traffic. This is a **clean correctness argument** — int32 accumulation cannot lose precision below int16 delta quantization error, and the RMSNorm fp32 scale absorbs the integer scale factor exactly. Not an approximation; a precision-preserving coordinate change that pins the residual to L2. Relies on no CF finding — purely a stack-primitive argument (L2 cache, int32 SIMD accumulation via AVX2 `vpaddd`, RMSNorm fp32 re-normalization at layer boundaries).

**Residency arithmetic.** Residual stream in L2 instead of DRAM: 224 KB vs 112 KB (bf16) per token. Memory saving: 0 (int32 is larger than bf16 per element). The gain is bandwidth locality: 28 × 2048 × 8 bytes (read+write bf16 from DRAM) = 448 KB/token DRAM traffic for residuals → replaced by 28 × 2048 × 8 bytes L2 traffic (same bytes, 26× faster bus). At 11.5 GB/s DRAM: 448 KB / 11.5 GB/s = 38.8 µs/token saved on residual traffic — small against the ~160 ms/token weight-loading cost at 1.7B bf16, but adds up at 4B+ scale. Primary value: the int32 accumulation path enables downstream ideas (per-channel overflow detection as a structural probe, delta-compression of int16 deltas exploiting CF2 small-delta structure) without any quality tradeoff.

**Novelty gloss.** Kill list: APND (run 005 A4) uses append-only log of deltas and found cos similarity doesn't guarantee small L2 delta magnitude — this idea avoids the delta-compression claim entirely; accumulator is just an integer register, not a compressed log. Published: llama.cpp uses int8 activation quantization but accumulates in fp32; ExLlamaV2 uses int4 weights with fp16 accumulation. Per-channel int32 residual accumulator held in L2 cache as the architectural primitive has no named entry. The structural argument: RMSNorm's normalization at each layer boundary resets the scale, making the int32 accumulator an exact representation regardless of layer depth.

**Smallest experiment.** Claim: on Qwen3-1.7B-Base, replacing bf16 residual accumulation with int32 (where each $\Delta_L$ is quantized to int16 before addition, and RMSNorm reads the int32 scaled by 2^{-15}) produces ΔNLL < 0.05 nats on WikiText-2 512 tokens. Implement in PyTorch: intercept each layer's output, cast to int16, add to running int32 accumulator, pass int32/32768.0 to RMSNorm. Runtime: ~25 min CPU. Go threshold: ΔNLL < 0.05 nats. NO-GO finding: int16 quantization of layer deltas is lossy at the residual scale → delta magnitudes exceed int16 range at some layers → those layers need int32 deltas → the constraint requires per-layer dynamic range measurement before deployment.

**Primary risk.** Layer deltas $\Delta_L$ may have large absolute magnitude at some layers even though cos(h_L, h_{L+1}) ≈ 0.99 — run 005 A4 already identified this risk. If ‖$\Delta_L$‖ overflows int16 range, the accumulator is corrupted. Mitigation: run the smallest experiment first; if any layer overflows, fall back to int32 deltas (still L2-resident, still faster than DRAM bf16) with a per-layer scale factor folded into the RMSNorm γ.

## A2-PERM: Permutation-Only Gauge — Weight-Reordering for Codebook Alignment

**Constraint:** The computation graph must algebraically simplify under head-permutation × channel-permutation symmetry simultaneously — inference must be expressible in a coordinate frame where both symmetries are fixed.

**Track B.** The trained transformer has two classes of permutation freedom that survive RMSNorm (unlike O(d) rotations which do NOT commute with per-channel γ). (1) **Head permutation**: the 16 attention heads in W_Q/W_K/W_V/W_O can be reordered without changing the function — each head is a block-diagonal unit, and summing the outputs is permutation-invariant. (2) **Channel permutation within each head's 128-dim subspace**: neurons within a head are not ordered; any permutation of the 128 head dimensions is a symmetry if applied consistently to W_Q, W_K, W_V row blocks and W_O column blocks. Critically: these are **permutation** symmetries (no rescaling), and permutations commute with RMSNorm trivially (RMSNorm is per-channel, so a permutation just reorders the γ values). The kill v2-S3-R004-001 (GFQO) is dead because O(d) rotation ≠ permutation. This idea uses only S_{16} × S_{128} (head and within-head permutations). The mechanism: apply a joint calibration-derived permutation to all 28 layers' attention blocks simultaneously so that head-0 through head-15 are sorted by decreasing "utilization" (e.g., mean attention entropy or W_Q row L2 norm). After sorting, heads 12–15 are the four least-used heads. A 4-bit codebook assigned to heads 0–11 and 2-bit codebook to heads 12–15 is now a **structured mixed-precision scheme** where the permutation alignment guarantees the low-precision heads are consistently the least load-bearing. Constraint difficulty: eliminates any scheme that assigns precision per-weight without structural alignment (~65% of standard mixed-precision approaches use gradient-based or activation-based sensitivity without this permutation alignment). The permutation is a one-time preprocessing step (no inference overhead).

**Residency arithmetic.** 4B model (IQ4_XS baseline, ~2.1 GiB). Attention weights are W_Q + W_K + W_V + W_O = 4 × 2048 × 2048 × 4 layers... for 1.7B model: 28 layers × (4 × 2048 × 2048) × 2 bytes = 28 × 4 × 4MB × 0.5 = 448 MB attention weights at bf16. At 4 bpw: 112 MB. If 4 of 16 heads are quantized to 2 bpw (25% of attention weights): 112 MB × (12/16 × 4bpw + 4/16 × 2bpw)/4bpw = 112 MB × 0.875 = 98 MB. Saving: 14 MB. Modest, but the permutation alignment makes the mixed-precision assignment provably structure-consistent — no ad-hoc sensitivity proxy needed. Combined with CF11 (global W_Q K=128 compression), the permuted+compressed W_Q saves additional bandwidth.

**Novelty gloss.** Kill list: v2-CHEAP-TEST-001 (cross-layer W_Q stacking) — this does NOT stack W_Q across layers; it permutes within-layer heads. v2-S3-R004-001 (GFQO arbitrary orthogonal rotation) — explicitly killed; this uses S_{16} × S_{128} not O(d). Published: SmoothQuant migrates scale across layers (not permutations); SpAtten prunes heads by importance score (no permutation-alignment story); A3 (arXiv:2505.12942) uses attention-logit-error for joint Q-K projection. Calibration-derived joint permutation for structured mixed-precision assignment has no named entry. The structural move: permutation symmetry is the ONLY gauge freedom in RMSNorm-normalized transformers that is both (a) a continuous group action and (b) exactly preserved, and it has not been exploited as a codebook-alignment primitive.

**Smallest experiment.** Claim: on Qwen3-1.7B-Base, the 16 attention heads ranked by mean W_Q row L2 norm have a 3:1 ratio between top-4 and bottom-4 head norms (i.e., the bottom-4 heads are structurally less active). Load W_Q for all 28 layers; for each layer compute per-head mean row L2 norm (mean over the 128 rows per head); sort heads by norm; measure ratio = mean(top-4 norms) / mean(bottom-4 norms) across all 28 layers. Runtime: <10 min. Go threshold: mean ratio > 2.0. NO-GO finding: all heads have similar norms → head permutation provides no alignment signal for mixed-precision assignment → the structural assumption is wrong, and head importance must be measured functionally (attention entropy, calibration loss contribution) rather than by weight norm.

**Primary risk.** Head importance measured by W_Q row norm may not correlate with actual attention quality contribution — a head with low W_Q norm might attend sharply to rare but critical tokens. Mitigation: if norm-based ranking fails GO, use calibration-pass attention entropy as the sorting key (still a permutation, just keyed differently); the structural argument for permutation-based mixed-precision alignment survives even if the specific ranking heuristic changes.

## A3-CADDR: Content-Addressed Residual Deduplication — Hash-Keyed KV Bypass

**Constraint:** Every weight is content-addressable by its action — no spatial layout; repeated residual-stream states that hash-collide are served from cache without re-running downstream layers.

**Track B / Track A (cross-track).** During autoregressive inference over a long context or repeated prompt structure, the residual stream $h_L^{(t)}$ at layer $L$ for token $t$ may be nearly identical to $h_L^{(t')}$ for some prior token $t' < t$. Under the content-addressable constraint: maintain a BLAKE3-keyed content-addressed cache of (layer, residual-state → logit-distribution) pairs. At each layer during decode, hash the current $h_L^{(t)}$ (quantized to int8 for hashing, 2048 bytes) and probe the cache. On hit: skip layers $L$ through $L_{\text{top}}$ and serve the cached logit distribution directly. Constraint difficulty: eliminates all spatial-layout assumptions, forces content-addressed memory management (~60% inadmissible for standard KV cache designs). Stack primitives: BLAKE3 hash (128-bit output, ~1 GB/s on Ryzen AVX2), a small in-RAM content-addressed skip table (e.g., 1M entries × 16 bytes key + 4 bytes pointer = 20 MB), NVMe-resident cached logit distributions for overflow. The mechanism differs from KV cache: standard KV cache stores (K, V) per layer per token; this stores (residual-state-hash → remaining-layer-output) and serves entire suffix-layer blocks on cache hit.

**Residency arithmetic.** Hash table in DRAM: 20 MB for 1M entries. Per cache hit: skip 14 layers (half the stack) = 14 × weight_bytes saved. At 1.7B weights at 4 bpw (~850 MB) split evenly: 14/28 × 850 MB = 425 MB not loaded per hit. Hit rate is the key unknown. For repetitive generation (code completion, template filling), even 5% hit rate: 0.05 × 425 MB / 11.5 GB/s = 1.8 ms/token saved on average. At 160 ms/token baseline: ~1.1% speedup. Interesting as a structural probe, not a primary residency lever at low hit rates. High hit rate (>30%) in repetitive domains: 30% × 425 MB / 11.5 GB/s = 11 ms saved = ~6.9% speedup. The content-addressable constraint forces measuring this hit rate empirically rather than assuming it.

**Novelty gloss.** Kill list: no content-addressed layer-skip entry. Published: key-value caching stores (K, V) per position; prompt caching in API inference stores KV prefix states at fixed positions (not content-addressed by residual hash). The closest published idea: speculative decoding with early-exit (Schuster et al. 2022, Confident Adaptive LM) exits early on confidence, not on residual-state hash collision. Content-addressed skip by residual hash has no named entry; the BLAKE3 key is the structural move that makes it testable.

**Smallest experiment.** Claim: on Qwen3-1.7B-Base generating 200 tokens from a repetitive prompt (e.g., repeating a 50-token template), the fraction of decode steps where $h_{14}^{(t)}$ (int8-quantized, BLAKE3-hashed) collides with a prior step's hash is > 2%. Run inference; at layer 14 each step, quantize h_14 to int8, compute BLAKE3 hash, check against a rolling set of prior hashes. Runtime: ~15 min. Go threshold: collision rate > 2% on repetitive prompt. NO-GO finding: residual states at layer 14 are unique per token even in repetitive context → content-addressed deduplication hit rate is negligible for LLM autoregressive decode → structural finding: the residual stream is token-specific enough that content-addressed bypass is not viable without much coarser (lossy) hashing.

**Primary risk.** Autoregressive residual states are nearly always token-unique because the residual stream encodes the full positional + causal context, not just the current token — hash collision rate is likely near zero. Mitigation: the NO-GO finding is itself useful (confirms residual-stream uniqueness); the experiment can be modified to test approximate collision (int4 quantization before hashing) to find the granularity at which collisions emerge.

## A4-MONO: Monotone Computation Graph — Only Non-Decreasing Functions at Each Layer

**Constraint:** The computation graph must consist only of monotone functions (componentwise non-decreasing in each input) throughout — no subtraction, no negative weights, no sign-flip operations.

**Track A.** Standard transformer arithmetic is emphatically non-monotone: residual subtraction is forbidden, negative weight entries are forbidden, softmax normalization involves division (non-monotone in the denominator). Under strict monotone constraint, the only admissible operations per-layer are: (a) componentwise max/min pooling, (b) componentwise multiplication by non-negative weights, (c) addition of non-negative quantities, (d) tropical operations (max-plus). This forces a radical reframing: the inference graph becomes a **monotone circuit** — a well-studied object in Boolean complexity theory. In the real-number analog (monotone real arithmetic circuits), representation power is bounded: monotone circuits cannot represent subtraction, so negative correlations between features are inexpressible. Constraint difficulty: eliminates standard attention (softmax involves QK^T with negative entries), standard FFN (negative weights), standard LayerNorm (subtracts mean) — ~95% inadmissible. What survives: (a) ReLU networks with non-negative weights (Nonnegative Matrix Factorization-style); (b) max-pooling-based attention; (c) tropical circuits. The binding constraint forces the question: **what fraction of a trained transformer's function can be expressed as a monotone circuit?** This is a falsifiable structural question about the sign distribution of learned weight products.

**Residency arithmetic.** A monotone-circuit version of a 1.7B transformer has no negative weights → all weights stored as unsigned integers → 1 bit saved per weight for the sign (which is always positive). At 4 bpw: dropping 1 sign bit → 3 bpw unsigned (magnitude only). 1.7B model at 4 bpw = 850 MB → at 3 bpw = 637 MB. Saving: 213 MB (25%). Reality: trained transformers have ~50% negative weights by random chance — the monotone constraint is NOT satisfied by the trained network. The experiment tests whether a post-hoc sign-projection (set negative weights to zero, renormalize) can be applied with ΔNLL < 1.0 nat. If yes, the 3 bpw unsigned representation is viable. If no, the monotone constraint is too tight for the trained weights and the structural finding is the negative-weight density.

**Novelty gloss.** Kill list: no monotone-circuit constraint entry. Published: NMF-based weight decomposition (killed by CF7/CF8 for MLP weights); ReLU-only networks (older GPT-2 style); tropical attention (A2-TROP run 005, Track A). Monotone circuit theory applied to transformer quantization has no named entry. The non-obvious move: treating the sign of weights as a free gauge bit that could be eliminated if the network were monotone — this is the constraint that forces the structural question about sign distribution.

**Smallest experiment.** Claim: on Qwen3-1.7B-Base, the fraction of W_up rows with all non-negative entries (after no adjustment) is < 5%, confirming the trained network is far from monotone. Load W_up layer 0; count rows with min value ≥ 0; report fraction. Runtime: 5 min. Go threshold (strong): > 30% non-negative rows (network is partially monotone, magnitude-only quantization worth pursuing). Go threshold (weak): > 5%. NO-GO (< 5%): network is fully non-monotone → monotone-circuit projection requires zeroing ~50% of weights per row → ΔNLL catastrophic → structural finding: trained Qwen3 weights require sign-preserving representations; magnitude-only storage is infeasible.

**Primary risk.** Zeroing negative weights changes the function catastrophically — this is not a sign-flip gauge (which preserves function) but a sign-truncation that destroys learned negative correlations. ΔNLL collapse is the expected outcome. Mitigation: frame as structural probe only (what is the negative-weight density?); the smallest experiment is the full payload regardless of go/no-go direction, since the sign density is useful for calibrating any sign-aware codebook design.

## A5-RDST: Reversible Decode Stack — Bennett-Style No-Erase Inference

**Constraint:** No information may be erased during inference — every intermediate activation is recoverable from the final output (Bennett reversible computation).

**Track A.** Bennett's reversible computation model (1973) shows any computation can be made reversible at the cost of storing "garbage" bits. For transformer inference, the constraint forces: each layer's output uniquely determines its input (layer function must be bijective on its domain). Standard transformers are NOT reversible: the SwiGLU gate $\text{silu}(W_{\text{gate}} x) \odot (W_{\text{up}} x)$ is not injective. Under the reversibility constraint, each layer must be replaced with a **coupling-layer** structure (as in real NVF / normalizing flows): split residual into two halves, $(a, b)$; update $a \leftarrow a + f(b)$, $b \leftarrow b + g(a)$ where $f, g$ are arbitrary (not required bijective themselves). This is the RevNet / i-RevNet structure. Constraint difficulty: eliminates all non-invertible activations (SwiGLU, softmax), all layer-norm with subtraction, all standard attention aggregation — ~90% inadmissible. What survives: coupling-layer residual networks, invertible 1×1 convolutions (Glow-style), affine coupling. The structural question: can the 28-layer Qwen3 residual stream be re-interpreted as a depth-28 normalizing flow? If yes, inference is equivalent to a flow evaluation, and the **inverse pass** is the generator (structured output sampling without autoregression). The Bennett constraint forces this question because it requires every forward-pass state to be recoverable from the final state — which is exactly the invertibility condition of a normalizing flow.

**Residency arithmetic.** Reversible inference at the coupling-layer level: store only $(h_0, h_{\text{final}})$ per token; intermediate states $h_1 \ldots h_{27}$ are not stored (recomputed on demand via inverse). Residency for K-token context: instead of KV cache (28 layers × K × 2048 × 2 bytes for K and V), store only final-layer residual (K × 2048 × 2 bytes = 4 MB at K=1000) plus coupling-layer parameters. KV cache is eliminated if attention can be reformulated as a coupling layer. At K=4096 tokens: KV cache = 28 × 4096 × 2 × 2048 × 2 bytes = 1.88 GB → coupling-layer inference stores 0 bytes for KV; regenerates K and V from final hidden state. This is only possible if Qwen3's attention is invertible — a strong structural claim that needs measurement.

**Novelty gloss.** Kill list: RECS (EMA recurrent KV) killed by RetNet/RADLADS prior art — different mechanism; RECS was recurrent state, this is invertible flow. Published: i-RevNet (Jacobsen et al. 2018) shows trained ResNets can be interpreted as coupling layers; RevNet (Gomez et al. 2017) training. None applied to LLM inference as a NO-KV-CACHE mechanism at test time. The structural claim that a trained transformer's residual stream is approximately a normalizing flow is testable via reconstruction error of $h_0$ from $h_{28}$ via pseudo-inverse pass and has no named v2 kill.

**Smallest experiment.** Claim: the pseudo-inverse reconstruction error ‖$\hat{h}_0 - h_0$‖_F / ‖$h_0$‖_F < 0.10 for the top-50% of Qwen3-1.7B layers when each layer is treated as a coupling-layer (split d=2048 into two halves, reconstruct input from output via $a \leftarrow a - f(\hat{b}), b \leftarrow b - g(\hat{a})$ with $f, g$ = zero). Note: with $f=g=0$ this tests whether each layer's function is approximately the identity on the split halves — if the layer output nearly equals input, reconstruction trivially works. Runtime: ~20 min. Go threshold: reconstruction error < 0.10. NO-GO: layers are NOT invertible-approximating → Bennett reversibility requires full activation storage → KV cache cannot be eliminated via flow inversion → structural finding: Qwen3 layers have high information creation (not preservation).

**Primary risk.** Transformers are emphatically not normalizing flows — SwiGLU gates destroy information and softmax is non-injective. Reconstruction error is expected to be high. Mitigation: frame as measuring the **information creation rate** per layer, which is a structural finding regardless of go/no-go; the reversibility constraint is the forcing function that produces this measurement.

## A6-TBLK: Table-Driven Block Decode — Inference as Dictionary Lookup on Token N-Grams [FREE SWING]

**Constraint:** The inference loop must fit in a finite lookup table — every forward pass is a table retrieval, not a neural-network evaluation.

**Track A.** The constraint is: given a context of N tokens, the output logit distribution is a table entry indexed by the N-gram hash. For N=4 (4-gram context), the table has $V^4 = 151936^4$ possible entries — astronomically large, so the table must be sparse (only observed N-grams are stored). Stack primitive: a sorted array of (N-gram-hash → logit-distribution-compressed) pairs in a B-tree-style layout on NVMe, accessed via binary search (O(log M) for M entries). This is **not** a neural network at inference — it is a retrieval system. Constraint difficulty: eliminates all autoregressive neural-network inference (~99% inadmissible). What survives: the trained model is used **offline** to populate the table (run inference on a large corpus, store output distributions for observed N-grams). The online inference step is a table lookup. The constraint forces the question: what fraction of inference queries on a target domain are served by N-gram table lookup vs. neural-network fallback? For low-entropy domains (code completion, template filling, FAQ answering), N=4 table coverage may be high. This is the Constraint-Alien version of N-gram speculative decoding (EMNLP 2025), but it inverts the framing: instead of using N-grams to speed up neural inference, it uses the neural model offline to build a table that replaces neural inference. The fallback is neural inference only when the N-gram is unseen.

**Residency arithmetic.** Table size: assume 100M observed 4-grams from a 10B-token corpus. Each entry: 16-byte hash key + 128-byte compressed logit distribution (top-100 logits as (token_id, logit) uint16 pairs = 4 × 100 = 400 bytes... too large). Compress to top-10 logits: 10 × (2+2) = 40 bytes + 16-byte key = 56 bytes per entry. 100M entries × 56 bytes = 5.6 GB on NVMe. Binary search on NVMe: at 3 GB/s sequential, a binary search over 100M sorted entries requires ~27 seeks × 4 KB page = 108 KB per lookup → at 700 MB/s random NVMe: 108 KB / 700 MB/s = 154 µs per lookup. That is slower than matmul-based token generation. FASTER path: B-tree with in-DRAM upper levels (top 20 B-tree levels × 4 KB × 2^{20} / 2^{20} = ~4 MB upper tree in DRAM) + NVMe leaf lookup = 1 NVMe access = 4 KB / 700 MB/s = 5.7 µs. At 5.7 µs: 175K tok/s — vastly faster than neural inference if table hit. Mixed model: 80% table hit at 175K tok/s + 20% neural fallback at 6 tok/s → effective throughput ≈ 60 tok/s × 0.8 + 6 × 0.2 = 49 tok/s. The table hit rate on the target domain is the key unknown.

**Novelty gloss.** Kill list: Branch-Predictor Token Trie (R3/S2) was killed for pre-emption by N-Gram Trie Speculative Decoding (EMNLP 2025) and SuffixDecoding (NeurIPS 2025). This idea is NOT N-gram speculative decoding (which accelerates neural inference); it is offline neural inference → table construction → table-only online inference with neural fallback. The structural inversion is the novel move: the neural model moves offline; the NVMe B-tree is the runtime system. No kill-list entry covers this framing.

**Smallest experiment.** Claim: on a 10K-token WikiText-2 segment, the 4-gram overlap between a 1M-entry table (populated from the training set) and the test set is > 20%. Construct a Python dict of 4-grams from the first 100M tokens of WikiText-103 training set; for each 4-gram in the 10K-token test segment, check dict membership; report coverage fraction. Runtime: <30 min. Go threshold: 4-gram coverage > 20% on test set. NO-GO: < 5% coverage → N-gram table lookup is not viable for general LLM inference → structural finding: natural language 4-gram distribution is too sparse for table-driven inference at practical table sizes → fallback is always neural.

**Primary risk.** WikiText-2 test set is low-entropy but general LLM targets (chat, reasoning) have near-zero 4-gram overlap with any fixed corpus — table hit rate in production is effectively zero. Mitigation: scope to a narrow domain (Python code completion, SQL generation) where N-gram repetition is high; the experiment measures domain-specific hit rate, not universal coverage; the result bounds the deployment scope of the mechanism.

## A7-INTLOG: Integer-Logarithm Arithmetic — All Weights as Powers of Two

**Constraint:** All weight values must be exact powers of two (i.e., stored as integer exponents in int8) — no multiplication, only bit-shift and addition.

**Track B.** Under the integer-logarithm constraint, each weight $w_{ij}$ is stored as $e_{ij} \in \{-127, \ldots, 127\}$ representing $\text{sign}(w_{ij}) \times 2^{e_{ij}}$. The GEMV $y_i = \sum_j w_{ij} x_j$ becomes $y_i = \sum_j \text{sign}(e_{ij}) \times (x_j \text{ ASR } |e_{ij}|)$ where ASR is arithmetic right-shift (integer division by power of 2). No floating-point multiplication; only shifts and additions. Constraint difficulty: eliminates all floating-point weight representations, all non-power-of-two quantization grids (INT4, INT8, FP4, FP8 with non-power-of-two centroids) — ~80% inadmissible. Forces: a quantization grid that is exactly the set $\{\ldots, -4, -2, -1, -0.5, -0.25, \ldots, 0.25, 0.5, 1, 2, 4 \ldots\}$. This is a subset of standard 8-bit floating-point (which has power-of-two exponents but non-integer significands). Stack primitive: `_mm256_srav_epi32` (AVX2 arithmetic right-shift), `_mm256_add_epi32` (AVX2 add) — both available on Ryzen 5 7530U, with throughput 1 cycle/256-bit register. No FMA unit needed. Effective bpw: 8 bits for exponent + 1 bit for sign = 9 bits/weight total, which is WORSE than FP8. BUT: the exponent is the only load-bearing storage; the sign can be separated into a 1-bit plane (1 bit/weight) + 7-bit unsigned exponent (7 bits/weight) = 8 bits total, matching INT8. The question is whether rounding all weights to the nearest power of two produces acceptable ΔNLL.

**Residency arithmetic.** 1.7B model at 9 bpw (8-bit exponent + 1-bit sign, packed as int16): ~3.4 GB. Worse than bf16. Useful bpw case: 7-bit unsigned exponent + 1-bit sign = 8 bpw = same as INT8. 1.7B at 8 bpw = 1.7 GB. At DRAM 11.5 GB/s: 1.7 GB / 11.5 GB/s × overhead ≈ 0.148 s/token → 6.8 tok/s (vs ~3 tok/s at bf16 8 bpw for reference). The compute win is real: shift+add vs FMA. Ryzen 5 7530U AVX2: 8 FMA/cycle vs 32 shift+add/cycle (4× throughput for shifts). But GEMV is bandwidth-limited, not compute-limited at this scale — so the compute gain is mostly irrelevant. Niche win: integer-logarithm arithmetic enables exact accumulation in int32 without rounding error from FMA, matching the A1-PCAG integer-accumulator idea.

**Novelty gloss.** Kill list: no logarithm-domain weight quantization entry. Published: LogQuant (arXiv:2407.02124, Jul 2024) stores activations in log domain; BinaryBERT uses ±1 weights; LUT-based methods (LUTGEMM, etc.) replace multiplications with lookups. Storing transformer weights as integer exponents (shift-only GEMV) has been proposed in hardware DSP contexts but has no named LLM deployment entry. The constraint is forcing: if multiplication is forbidden, the only linear map is shift+add, and the only weight representation compatible with that is the logarithmic grid.

**Smallest experiment.** Claim: on Qwen3-1.7B-Base, rounding all W_up weights to the nearest power of two (int8 log-quantization: $\hat{w} = \text{sign}(w) \times 2^{\text{round}(\log_2 |w|)}$) achieves ΔNLL < 2.0 nats on WikiText-2 512 tokens. Load W_up layer 0, bf16; apply log-quantization; compute forward pass on 512 tokens with quantized W_up, all other weights at bf16; report ΔNLL. Runtime: <20 min. Go threshold: ΔNLL < 2.0 nats. NO-GO finding: log-quantization of W_up is catastrophically lossy → weight distribution is not log-uniform → nearest-power-of-two rounding is a poor quantization grid for trained transformers → structural finding: weight distributions require finer sub-logarithmic quantization grids, not power-of-two only.

**Primary risk.** Trained transformer weights (per CF4/CF5) are full-rank and have non-log-uniform magnitude distributions — the nearest-power-of-two grid is extremely coarse near zero, where most weights cluster. ΔNLL will likely exceed 2.0 nats. Mitigation: the NO-GO finding is the useful structural result (weight magnitude distribution deviates strongly from log-uniform → motivates floating-point grid quantization rather than integer-exponent), which is itself a CF-level structural finding for the v2 ladder.

## Convergence handles

- **Per-channel int32 residual accumulation held in L2 cache** — A1-PCAG; cross-orients with any idea that reads residual-stream delta magnitudes (C / F orientations needing ‖$\Delta_L$‖ vs ‖$h_L$‖)
- **Permutation-group gauge of attention heads** — A2-PERM; requires head-utilization ranking (W_Q row norms or attention entropy); convergence signal if Composition or First-Principles independently surface head-permutation structure
- **Content-addressed skip of layer suffix blocks** — A3-CADDR; requires measuring residual-state collision rate at int8/int4 quantization; cross-orients with any KV-side or early-exit idea
- **Sign/negative-weight density of trained weights** — A4-MONO and A7-INTLOG both need this measurement; any future sign-aware codebook design depends on it; not yet a confirmed CF
- **Reversibility / information creation rate per layer** — A5-RDST; requires measuring reconstruction error ‖$\hat{h}_0 - h_0$‖ per layer; relevant to any information-bottleneck or flow-based proposal across orientations
- **N-gram table coverage on target domain** — A6-TBLK; requires domain-specific 4-gram overlap measurement; convergence signal if Unconventional Substrate independently proposes NVMe B-tree retrieval
